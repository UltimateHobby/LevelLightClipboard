Level Light Clipboard User Guide

Version 1.0.0 UE 5.6 / 5.7 Windows

1. Overview

Level Light Clipboard copies native Unreal Engine lighting settings from one level into an editable DataAsset, then pastes selected modules into another level.

The tool is designed for deliberate editor work. Before copying or pasting, it shows the lighting actors involved so you can choose the exact source or target when a level contains multiple compatible actors.

2. Requirements

  • Unreal Engine 5.6 or 5.7 on Windows.
  • A plugin package built for the same Unreal Engine version you are opening.
  • A level that uses Unreal's native lighting actors.
  • Blueprint-only projects are supported when using the packaged plugin build.

3. Install

  1. Close Unreal Editor.
  2. Copy the matching plugin package into one of these locations:
<Project>/Plugins/UE5LevelLightCopyPaste
<UE_5.x>/Engine/Plugins/Marketplace/UE5LevelLightCopyPaste
  1. Open the project in the matching Unreal Engine version.
  2. Enable Level Light Clipboard if Unreal prompts you.
  3. Restart Unreal Editor if requested.

4. Open the Tool

In Unreal Editor, open the main menu item:

Tools -> Lighting Clipboard

The Lighting Clipboard panel should appear as an editor tab.

5. Copy Lighting

  1. Open the source level.
  2. Click Copy From Current Level.
  3. Review every detected lighting module in the dialog.
  4. Uncheck modules you do not want to capture.
  5. Choose a different compatible source actor if the selected actor is not the one you want.
  6. Capture into a new or existing Lighting Clipboard DataAsset.

6. Paste Lighting

  1. Open the target level.
  2. Select the source Lighting Clipboard DataAsset.
  3. Click Paste Into Current Level.
  4. Check only the modules you want to apply.
  5. Choose the destination actor for each checked module.
  6. If a target actor is missing, choose whether to create it or skip that module.
  7. Confirm the paste.

Paste operations are undoable through Unreal Editor's normal undo stack.

7. Supported Modules

  • Sun / primary Directional Light
  • Optional moon / secondary Directional Light
  • Sky Light
  • Sky Atmosphere
  • Exponential Height Fog
  • Volumetric Cloud
  • Post Process Volume
  • Selected World Settings fields

8. Lighting Clipboard DataAssets

Captures are stored as editable DataAssets. You can inspect and adjust structured lighting values such as intensity, color, Sky Light source settings, fog density, cloud settings, and post-process overrides.

The plugin also stores hidden snapshots to preserve engine settings that are not exposed as primary editable fields. These snapshots improve paste fidelity while keeping the normal editing surface simple.

9. Troubleshooting

Problem Check
The plugin does not appear. Confirm the plugin package matches the Unreal Engine version you opened.
Unreal asks to rebuild the plugin. Use the packaged build for your engine version, or open the source package in a C++-capable setup.
A lighting actor is marked as missing. Choose a compatible actor manually, create the missing actor during paste, or skip that module.

10. Boundaries

  • The plugin focuses on Unreal's native lighting actors.
  • Ultra Dynamic Sky, Sky Creator, and project-specific environment rigs are not copied directly.
  • The plugin does not add a runtime day/night or weather system.
  • The plugin does not copy project settings, console variables, or destructive scene cleanup rules.
  • World Settings coverage is limited to selected lighting and render fields.
