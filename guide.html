User guide

Level Light Clipboard copies engine-native lighting setups from one Unreal Engine level into an editable DataAsset, then pastes selected modules into another level.

Open the tool

Tools -> Lighting Clipboard

Supported modules

  • Sun / primary Directional Light
  • Optional moon / secondary Directional Light
  • Sky Light
  • Sky Atmosphere
  • Exponential Height Fog
  • Volumetric Cloud
  • Post Process Volume
  • Selected World Settings fields

Editable DataAssets

Clipboard DataAssets expose the structured fields artists usually want to edit directly, such as intensity, color, fog density, cloud settings, and post-process overrides. Hidden exact snapshots help preserve fidelity while the structured fields remain the normal editing surface.

Copy lighting

  1. Open the source level.
  2. Click Copy From Current Level.
  3. Review each detected lighting module.
  4. Uncheck anything you do not want to capture.
  5. If needed, choose another compatible source actor in that row.
  6. Capture into a new or existing clipboard DataAsset.

Paste lighting

  1. Open the target level.
  2. Click Paste Into Current Level.
  3. Choose the source clipboard DataAsset.
  4. Check only the modules you want to apply.
  5. Choose the destination actor for each module, or explicitly create or skip missing targets.
  6. Confirm the paste.

Paste is undoable through the normal Unreal Editor undo stack.

Installation

Use the packaged build that matches your Unreal Engine version.

<Project>/Plugins/UE5LevelLightCopyPaste

<UE_5.x>/Engine/Plugins/Marketplace/UE5LevelLightCopyPaste

  1. Close Unreal Editor before copying the plugin into the project or engine plugin folder.
  2. Reopen the project in the matching Unreal Engine version.
  3. If Unreal prompts you to enable Level Light Clipboard, confirm and restart the editor.
  4. Open Tools -> Lighting Clipboard and verify that the panel opens.
  5. Click Copy From Current Level and verify that the module-selection dialog appears.
Blueprint-only projects are supported when using the packaged build.

Troubleshooting

  • If the plugin does not appear, confirm that the package version matches the Unreal Engine version you opened.
  • If Unreal asks to rebuild the plugin, use a source-capable environment or install the packaged build that matches your engine version.
  • If a lighting actor is marked as missing during copy or paste, choose a compatible actor manually or create the missing destination actor explicitly in the paste dialog.

Known boundaries

  • Version 1.0.0 focuses on Unreal's native lighting actors.
  • Ultra Dynamic Sky, Sky Creator, and project-specific environment rigs are not copied directly.
  • The plugin does not copy project settings, console variables, or destructive cleanup rules.
  • World Settings coverage is intentionally limited to selected lighting and render fields.
